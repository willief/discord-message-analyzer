DATE RANGE SPECIFICATION
================================================================================

START_DATE: 2024-02-01T00:00:00Z
└─ Fixed date: February 1, 2024 at 00:00:00 UTC

END_DATE: Dynamic (System time - 1 hour)
├─ Calculation: date -u --date='1 hour ago' '+%Y-%m-%dT%H:%M:%SZ'
├─ Current example: 2024-11-19T16:30:45Z (if now is 17:30:45 UTC)
└─ Purpose: Ensure data freshness and stability

USAGE EXAMPLE:
- Current time (UTC): 2024-11-19 17:30:00
- End date (- 1 hour): 2024-11-19 16:30:00
- Full range: 2024-02-01T00:00:00Z to 2024-11-19T16:30:00Z

================================================================================
DATE FORMAT REQUIREMENTS
================================================================================

Format: ISO 8601 with UTC timezone
Syntax: YYYY-MM-DDTHH:MM:SSZ

Valid Examples:
  ✓ 2024-02-01T00:00:00Z    (February 1, 2024 midnight)
  ✓ 2024-11-19T16:30:45Z    (November 19, 2024 at 16:30:45)
  ✓ 2024-06-15T12:00:00Z    (June 15, 2024 noon)

Invalid Examples:
  ✗ 2024-02-01              (missing time)
  ✗ 02-01-2024              (wrong order)
  ✗ 2024-02-01 12:00:00     (missing Z)
  ✗ 2024-02-01T12:00:00+00:00 (use Z not +00:00)

TECHNICAL DETAILS
================================================================================

Processing Flow:

  generate_multi_user_digest.sh
  ├─ Reads: ./digests/*.json
  ├─ Calculates: END_DATE = now - 1 hour
  └─ For each user:
     ├─ Calls: discord_digest.py with date parameters
     └─ discord_digest.py
        ├─ Loads: All messages from JSON exports
        ├─ Filters:
        │  ├─ Username (exact match, lowercase)
        │  ├─ Date range (START_DATE ≤ message ≤ END_DATE)
        │  └─ Replies to username
        ├─ Organizes: By date and type
        └─ Outputs:
           ├─ Complete archive (all messages)
           ├─ Weekly digest (14 days)
           ├─ HTML (styled)
           └─ JSON (structured)

Date Filtering Logic (Python):
  if self.date_from:
      if message_datetime < parse(date_from):
          skip_message()
  
  if self.date_to:
      if message_datetime > parse(date_to):
          skip_message()

DateTime Handling:
  - Input: ISO 8601 with Z (2024-02-01T00:00:00Z)
  - Processing: Convert Z to +00:00 for Python parsing
  - Storage: Python datetime objects (timezone-aware)
  - Comparison: Direct datetime comparison (safe)
